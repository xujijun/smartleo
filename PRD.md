# SmartLeo 儿童学习 PWA 需求文档

## 1. 项目概述

`SmartLeo` 是一个面向儿童早教场景的移动端学习应用，采用 PWA（Progressive Web App，渐进式 Web 应用）形态，支持用户通过浏览器菜单“添加到主屏幕”安装到手机桌面，获得接近原生应用的使用体验。

- 项目名称：`smartleo`
- 部署地址：<https://xujijun.github.io/smartleo>
- 部署方式：GitHub Pages + GitHub Actions 自动化部署

## 2. 建设目标

- 提供一个轻量、易用、适合儿童的学习入口页。
- 支持至少两类基础学习内容：英文字母学习、简单汉字学习。
- 通过 PWA 提升移动端访问体验（可添加到主屏幕）。
- 建立可持续迭代的前端工程体系（Vue 3 + Vite + TypeScript + Vant）。

## 3. 目标用户与使用场景

### 3.1 目标用户

- 主要用户：学龄前及小学低年级儿童。
- 辅助用户：家长（陪同学习、引导操作）。

### 3.2 核心场景

- 家长在手机浏览器打开应用。
- 首次进入首页，选择学习入口（英文字母/简单汉字）。
- 在学习页面完成基础认知学习（识别、跟读或浏览）。
- 通过“添加到主屏幕”将应用安装至桌面，后续快速进入。

## 4. 产品范围（MVP）

## 4.1 首页

- 展示应用名称与简要欢迎信息。
- 提供多个学习入口卡片（至少包含）：
  - 英文字母学习
  - 简单汉字学习
- 入口卡片需适配移动端触控交互，点击后跳转对应页面。

### 4.2 英文字母学习模块

- 展示 A-Z 字母列表（大小写可同时展示）。
- 支持逐个字母查看（如卡片切换/列表点击）。
- 每个字母需有基础认知信息（字母形态，后续可扩展发音）。

### 4.3 简单汉字学习模块

- 展示常见简单汉字列表（示例：一、二、三、人、口、日、月等）。
- 支持逐个汉字查看。
- 每个汉字需有基础认知信息（字形展示，后续可扩展拼音/释义）。

### 4.4 PWA 能力

- 提供 `manifest`，配置应用名称、图标、启动显示方式等。
- 支持 HTTPS 环境下浏览器“添加到主屏幕”。
- 注册 `service worker`（至少满足基础静态资源缓存能力）。

## 5. 非功能性需求

### 5.1 技术栈要求

- 前端框架：Vue 3
- 构建工具：Vite
- UI 组件库：Vant
- 语言：TypeScript
- 包管理工具：推荐 npm（可按团队规范使用 pnpm/yarn）

### 5.2 兼容性与终端要求

- 以移动端浏览器为主（iOS Safari、Android Chrome）。
- 页面采用响应式布局，优先适配主流手机尺寸。
- 首屏加载与交互应流畅，避免明显卡顿。

### 5.3 可维护性要求

- 目录结构清晰，按页面/模块组织。
- 路由与页面组件分离，便于扩展更多学习入口。
- 关键逻辑与配置具备基础注释说明。

## 6. 信息架构与页面流转

- `/`：首页（学习入口聚合）
- `/letters`：英文字母学习
- `/hanzi`：简单汉字学习

页面流转：
1. 用户进入首页 `/`
2. 点击入口进入 `/letters` 或 `/hanzi`
3. 在模块内学习后，可通过导航返回首页继续选择

## 7. UI/交互需求（移动端）

- 风格：简洁、明快、儿童友好（高可读性、适度留白）。
- 首页入口：卡片式布局，点击反馈明确。
- 字体：中文/英文均保证清晰可读，字号不宜过小。
- 操作路径：尽量 1-2 次点击到达学习内容，减少复杂操作。

## 8. 部署与发布要求

### 8.1 GitHub Pages

- 仓库启用 GitHub Pages，发布目录为构建产物目录（通常为 `dist`）。
- 对外访问 URL 固定为：<https://xujijun.github.io/smartleo>

### 8.2 GitHub Actions 自动化部署

- 在代码 push 到指定分支（如 `main`）后自动触发工作流。
- 工作流执行步骤至少包括：
  - 安装依赖
  - 执行构建（`vite build`）
  - 将构建结果部署到 GitHub Pages
- 部署失败时应能在 Actions 日志中定位原因。

## 9. 验收标准

满足以下条件可判定 MVP 验收通过：

1. 访问 <https://xujijun.github.io/smartleo> 可正常打开首页。
2. 首页至少存在“英文字母学习”“简单汉字学习”两个入口并可跳转。
3. 两个学习模块均可展示基础学习内容且无明显功能错误。
4. 应用支持浏览器“添加到主屏幕”。
5. 代码推送后可通过 GitHub Actions 自动部署到 GitHub Pages。

## 10. 迭代建议（可选）

- 增加发音能力（字母音频、汉字读音）。
- 增加学习进度与打卡机制。
- 增加互动练习（选择题/连线题/听音识字）。
- 增加离线学习增强策略（更完整缓存与更新提示）。

# SmartLeo 儿童学习 PWA 需求文档

## 1. 项目概述

`SmartLeo` 是一个面向儿童早教场景的移动端学习应用，采用 PWA（Progressive Web App，渐进式 Web 应用）形态，支持用户通过浏览器菜单“添加到主屏幕”安装到手机桌面，获得接近原生应用的使用体验。

- 项目名称：`smartleo`
- 部署地址：<https://xujijun.github.io/smartleo>
- 部署方式：GitHub Pages + GitHub Actions 自动化部署

## 2. 建设目标

- 提供一个轻量、易用、适合儿童的学习入口页。
- 支持至少三类基础学习内容：英文字母学习、简单汉字学习、字母测试。
- 通过 PWA 提升移动端访问体验（可添加到主屏幕）。
- 建立可持续迭代的前端工程体系（Vue 3 + Vite + TypeScript + Vant）。

## 3. 目标用户与使用场景

### 3.1 目标用户

- 主要用户：学龄前及小学低年级儿童。
- 辅助用户：家长（陪同学习、引导操作）。

### 3.2 核心场景

- 家长在手机浏览器打开应用。
- 首次进入首页，选择学习入口（英文字母/简单汉字/字母测试）。
- 在学习页面完成基础认知学习（识别、跟读或浏览）。
- 在字母测试模块中完成 20 道听音选字母题，并查看最终分数与语音结果反馈。
- 通过“添加到主屏幕”将应用安装至桌面，后续快速进入。

## 4. 产品范围（MVP）

## 4.1 首页

- 展示应用名称与简要欢迎信息。
- 提供多个学习入口卡片（至少包含）：
  - 英文字母学习
  - 简单汉字学习
  - 字母测试
- 入口卡片需适配移动端触控交互，点击后跳转对应页面。
- 首页主要可点击区域（如学习入口卡片、开始学习按钮）在点击后应提供轻量声音提示，增强儿童交互反馈。

### 4.2 英文字母学习模块

- 展示 A-Z 字母列表（大小写可同时展示）。
- 支持逐个字母查看（如卡片切换/列表点击）。
- 每个字母需有基础认知信息（字母形态、示例单词、基础发音）。
- 支持在适当交互中播放字母发音，如点击当前字母卡片、播放按钮或切换到新字母后播放对应读音。
- 用户点击字母切换按钮、上一页/下一页按钮时，应播放简短声音提示；如后续接入字母发音，需与普通点击提示音区分。

### 4.3 简单汉字学习模块

- 展示常见简单汉字列表（示例：一、二、三、人、口、日、月等）。
- 支持逐个汉字查看。
- 每个汉字需有基础认知信息（字形展示、拼音、基础读音）。
- 支持在适当交互中播放汉字读音，如点击当前汉字卡片、播放按钮或切换到新汉字后播放对应读音。
- 用户点击汉字切换按钮、上一页/下一页按钮时，应播放简短声音提示；如后续接入汉字读音，需与普通点击提示音区分。

### 4.4 字母测试模块

- 首页提供“字母测试”入口，进入后先展示测试说明和一个“开始测试”按钮。
- 点击“开始测试”后，系统生成 20 道题目，每道题随机抽取一个英文字母作为正确答案。
- 每道题开始时，页面使用语音朗读“请选择字母 X”，停顿 10 秒后再朗读一次相同内容。
- 每道题提供 20 秒倒计时，用户需在倒计时结束前从字母表中点击正确字母。
- 回答正确计 5 分，回答错误或超时计 0 分；回答或超时后，语音提示“正确”或“错误”后再进入下一题。
- 每题结束后自动进入下一题，直到 20 道题全部完成。
- 测试结束后展示总分统计结果，并通过动画方式强化结果反馈。
- 结果页需语音播报最终成绩，并根据得分追加鼓励语：
  - 100 分：播报加上“你是最棒的”
  - 90 到 95 分：播报加上“你很棒”
  - 60 到 85 分：播报加上“有点厉害”
  - 低于 60 分：播报加上“要加油哦”
- 测试过程中应有清晰的题号、当前分数、倒计时状态展示。
- 测试模块中的开始按钮、选项点击、答题结果切换等关键操作应保留轻量声音提示。

### 4.5 PWA 能力

- 提供 `manifest`，配置应用名称、图标、启动显示方式等。
- 支持 HTTPS 环境下浏览器“添加到主屏幕”。
- 注册 `service worker`（至少满足基础静态资源缓存能力）。

## 5. 非功能性需求

### 5.1 技术栈要求

- 前端框架：Vue 3
- 构建工具：Vite
- UI 组件库：Vant
- 语言：TypeScript
- 包管理工具：推荐 npm（可按团队规范使用 pnpm/yarn）

### 5.2 兼容性与终端要求

- 以移动端浏览器为主（iOS Safari、Android Chrome）。
- 页面采用响应式布局，优先适配主流手机尺寸。
- 首屏加载与交互应流畅，避免明显卡顿。

### 5.3 可维护性要求

- 目录结构清晰，按页面/模块组织。
- 路由与页面组件分离，便于扩展更多学习入口。
- 关键逻辑与配置具备基础注释说明。

## 6. 信息架构与页面流转

- `/`：首页（学习入口聚合）
- `/letters`：英文字母学习
- `/hanzi`：简单汉字学习
- `/letter-test`：字母测试

页面流转：
1. 用户进入首页 `/`
2. 点击入口进入 `/letters`、`/hanzi` 或 `/letter-test`
3. 在学习模块内学习后，可通过导航返回首页继续选择
4. 在字母测试模块中点击“开始测试”进入答题流程
5. 完成 20 题后进入结果页查看分数与语音反馈

## 7. UI/交互需求（移动端）

- 风格：简洁、明快、儿童友好（高可读性、适度留白）。
- 首页入口：卡片式布局，点击反馈明确。
- 字体：中文/英文均保证清晰可读，字号不宜过小。
- 操作路径：尽量 1-2 次点击到达学习内容，减少复杂操作。
- 对儿童高频点击的关键交互增加声音反馈，提示音应短促、柔和、不刺耳，避免造成打扰。
- 应提供基础声音策略：默认开启提示音，后续可扩展静音开关。
- 声音提示应在用户手势触发后播放，兼容移动端浏览器自动播放限制。
- 字母发音、汉字读音应清晰、简短，优先保证儿童可辨识度。
- 发音播放按钮或点击发音区域需有明确视觉提示，避免与普通页面切换操作混淆。
- 若因浏览器权限或网络原因导致发音资源播放失败，页面应尽量不影响主流程使用。
- 字母测试模块需突出“开始测试”“当前第几题”“剩余倒计时”“当前分数”等关键信息，避免儿童操作迷失。
- 倒计时展示应醒目，建议在剩余 2 秒内加入更明显的视觉提醒。
- 测试结果页应具备较强的结果反馈感，可通过数字动画、颜色变化或插画强化鼓励氛围。
- 结果播报文案需与分数区间严格对应，避免出现区间遗漏或重叠。

## 8. 部署与发布要求

### 8.1 GitHub Pages

- 仓库启用 GitHub Pages，发布目录为构建产物目录（通常为 `dist`）。
- 对外访问 URL 固定为：<https://xujijun.github.io/smartleo>

### 8.2 GitHub Actions 自动化部署

- 在代码 push 到指定分支（如 `main`）后自动触发工作流。
- 工作流执行步骤至少包括：
  - 安装依赖
  - 执行构建（`vite build`）
  - 将构建结果部署到 GitHub Pages
- 部署失败时应能在 Actions 日志中定位原因。

## 9. 验收标准

满足以下条件可判定 MVP 验收通过：

1. 访问 <https://xujijun.github.io/smartleo> 可正常打开首页。
2. 首页至少存在“英文字母学习”“简单汉字学习”“字母测试”三个入口并可跳转。
3. 英文字母模块支持基础字母发音播放，简单汉字模块支持基础读音播放。
4. 字母测试模块可正常开始测试，并按规则完成 20 道随机题目。
5. 字母测试模块中每题会语音播报“请选择字母 X”，并在 10 秒后重复一次。
6. 每题答题时间为 20 秒，正确计 5 分，错误或超时计 0 分，最终分数统计准确。
7. 测试完成后可展示结果动画，并按得分区间播报对应鼓励语。
8. 学习模块均可展示基础学习内容且无明显功能错误。
9. 首页及学习模块中的关键点击操作具备正常的声音提示反馈。
10. 应用支持浏览器“添加到主屏幕”。
11. 代码推送后可通过 GitHub Actions 自动部署到 GitHub Pages。

## 10. 迭代建议（可选）

- 增加发音能力（字母音频、汉字读音）。
- 增加学习进度与打卡机制。
- 增加互动练习（选择题/连线题/听音识字）。
- 增加测试历史记录与错题回顾。
- 增加离线学习增强策略（更完整缓存与更新提示）。
